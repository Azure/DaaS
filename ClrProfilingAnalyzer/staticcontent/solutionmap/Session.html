This request is stuck in the ASP.NET SessionState Module. This can happen under the following conditions:
  1. Multiple requests came for the same session ID. When this happens ASP.NET serializes the requests and executes only one request at a given point of time. This happens most commonly if your Web Application is making AJAX calls to web pages or Web API in the same application OR if you are running a stress test and you end up passing the same Session ID in all the requests.
  2. You are storing session data in an OutProc session store (for e.g. a SQL Server or Redis Cache) and the SessionState provider is introducing a delay.
Recommendations
  1. Evaluate if there is session state required for your application
  2. For pages not modifying Session State, mark the EnableSessionState property at the Page level to ReadOnly. For more details check this article.
  3. If you are using an OutProc Session state provider, then check if the provider is behaving normally
