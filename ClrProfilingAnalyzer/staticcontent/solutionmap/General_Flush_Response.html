The request spent most of the time in flushing out the response buffer. This can occur in the following cases:
  1. The application is returning too much data back from the server to the client. Check the sc-bytes property of this request to understand if that's the case (A zero value for sc-bytes indicates that the request has not finished in this trace yet so you might have to check if response data will be large for this request.).
  2. The Client is behind a slow network (for e.g. a mobile client) or in a different geography or geo-location.
  3. There is network latency between the client and this application.
